Current kod:

public class Main {

    private Application application;
    private JButton b1, b2, b3;

    class ActionHandler implements ActionListener {
        public void actionPerformed(ActionEvent e) {
            if (e.getSource() == b1) {
                application.action1();
            } else if (e.getSource() == b2) {
                application.action2();
            } else {
                application.action3();
            }
        }
    }
}

De principer som koder bryter är SRP (Single Responsibility Principle) eftersom att ActionHandler måste veta vilken knapp som tryckts 
och vilken metod på Application som ska anropas, koden bryter också mot OCP (Open/Closed Principer) eftersom att man inte på ett 
enkelt sätt kan lägga till nya class/actions utan ändra på andra ställern i koden. Den sista är att koden har hög Coupling,
vilket innebär i detta fallet att ActionHandler är hårt kopplat till Application och specifika metoder. 

Problemet leder till att koden är svår att utöka och underhålla.
